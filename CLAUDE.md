# 飛島ホールディングス サステナビリティページ改修ガイドライン

## 概要

以下ページ群のデザイン改修を行う。

対象はサステナビリティ配下の18ページ。

作業は Claude Code を使用し、
Figma MCP 連携によりデザインデータを参照しながら実装すること。

---

# 実装の基本方針

今回の改修は、
既存ページを一から作り直すものではない。

既存HTML・既存コンテンツ・共通パーツを活かしながら、
ページ内コンテンツ部分のみをデザイン改修する。

---

# 最重要事項

## 既存コンテンツを活かすこと

既存の以下は原則そのまま使用する。

- 見出し
- 本文
- テキスト内容
- リンク内容

### NG

- テキストリライト
- 要約
- 勝手なコピー追加
- 見出し変更
- コンテンツ内容変更

### OK

- 改行調整
- span追加
- wrapper追加
- レイアウト調整
- 装飾目的のマークアップ追加

---

# 改修範囲

## 変更対象

- メインコンテンツエリア
- ページタイトル周り
- リード文
- セクション見出し
- 本文レイアウト
- カードUI
- テーブル
- 画像周り
- CTA
- 下層導線
- ページ内ナビゲーション

---

## 絶対に変更しない箇所

以下は原則そのまま使用すること。

- header
- footer
- グローバルナビ
- ハンバーガーメニュー
- 共通メニュー
- 共通JS
- サイト全体のベース構造
- `<head>` 内の共通設定

header / footer を含めた全面改修は禁止。

# Figmaについて

Figma MCP を使用してデザインを読み込み、
デザインを参考に実装すること。

ただし、
「Figma完全再現」ではなく、
既存サイトとの統一感・保守性を優先する。

---

# 見出しルール

## メイン見出しについて

ページ内の主要セクション見出しは、
既存サイトとの統一感を維持するため、
原則 `hstyle1` を使用すること。

---

## 使用例

```html
<h2 class="hstyle1">環境への取り組み</h2>
```

# 既存マークアップ維持ルール

今回の改修では、
既存サイトで使用されている基本マークアップ構造を可能な限り維持すること。

特に、
既存ページで共通化されている以下構造は、
原則崩さず使用する。

```html
<section class="insec" id="sec1">
  <div class="inner">
    <h2 class="hstyle1"><span>環境への取組み</span></h2>
    <p class="text">...</p>
  </div>
</section>
```
